# 댓글 피드백 기능 UI 연동 계획

## 작업명
댓글 피드백 조회용 UI를 상호작용 가능한 버튼 기능으로 대체

## 목표
현재 단순 텍스트로 조회만 가능한 댓글의 좋아요/싫어요 피드백 UI를, 미리 정의된 API와 연동된 상호작용 가능한 버튼으로 변경하여 사용자가 실제로 피드백 기능을 이용할 수 있도록 한다.

## 근거 문서
- `../shared/requirements.md`: "각각의 댓글에는 피드백을 남길 수 있다. (like/dislike)"
- `../shared/api-spec.md`: "6. 추천 댓글 피드백", "10. 구매 과자 댓글 피드백" 엔드포인트 명세

## 선행 조건
- 추천 댓글 및 구매 과자 댓글 피드백을 위한 프론트엔드 통신 함수(`lib/request.ts` 내부 POST API) 유무 확인. 없으면 추가 필요.

## 작업 범위
- `frontend/lib/request.ts`: 추천 댓글 피드백 및 구매 과자 댓글 피드백 API 호출을 위한 axios 함수 추가 (누락되어 있을 경우)
- `frontend/components/CommentSection.tsx` 등의 댓글 목록 UI: 댓글 항목 하단의 피드백 표시 부분(👍 / 👎)을 onClick 이벤트 연동 버튼 구조로 변경하고, 내부 state(또는 props를 통한 상위 갱신)를 통해 토글 갱신되도록 처리.

## 제외 범위
- 추천 게시글 혹은 구매한 과자 제목/본문에 속한 '게시글 피드백' 기능의 수정은 본 작업에 포함하지 않음.

## 세부 정책
- **권한 확인:**
  - 사용자(USER)만 피드백 API 호출 가능.
  - 관리자(ADMIN)는 요구사항 정책상 피드백 불가이므로, 클릭 이벤트를 막거나 권한 없음을 알림 창으로 노출해야 함.
  - 비로그인 유저는 피드백 이용을 제한하도록 처리(로그인 유도 모달/알림 노출 등).
- **입력값/응답값:**
  - 입력값: `status` ("LIKE" 또는 "DISLIKE")
  - 출력값/응답: 서버 응답 DTO에서 반환한 `likeCount`, `dislikeCount`, `myFeedback` 최신 데이터를 바탕으로 해당 댓글의 UI 상태를 명확히 갱신.
- **실패 케이스:**
  - API 호출 중 오류 발생 시(서버 오류 500, 권한 오류 403 등) 이전 정보를 유지하며 사용자에게 alert 등을 통해 문제 안내.

## 완료 조건
- [ ] 일반 USER 계정 로그인 시, 각 추천 댓글 또는 구매 과자 댓글의 좋아요/싫어요 버튼을 눌렀을 때 해당 피드백이 토글되고(api_spec 기준), 정상적으로 화면에서 카운트가 변경된다.
- [ ] 동일 피드백 객체에 대해 다시 클릭 시 취소가, 다른 피드백 변경 시 토글 반영이 API 응답에 맞춰 화면에 즉각적으로 표기된다 (`myFeedback` 표기).
- [ ] ADMIN 계정이나 로그인하지 않은 사용자는 버튼을 눌렀을 때 권한 거부 관련 알림이 제대로 뜨는지, API 500 에러 등을 유발하지 않는지 확인된다.

## 미정 사항
- 사용자가 현재 어떤 피드백(LIKE / DISLIKE)을 주었는지 UI 적으로 어떻게 구분해서 보여줄 것인가(예: 활성화된 버튼에 갈색 배경 추가 등) 명시적 디자인이 정의되지 않았으므로 프론트엔드에서 직관적인 방식으로 반영해야 함.

## generator 전달 메모
- 댓글 피드백 API는 `POST /api/recommendations/{rcId}/comments/{rccId}/feedback` 이며 Body 포맷은 `{ "status": "LIKE" | "DISLIKE" }`입니다(구매 과자의 경우도 URL 경로 다르고 형식 동일).
- 프론트엔드 `CommentSection` 내부에 각 댓글 아이템 별로 상태를 별도 관리할지, 아니면 상위 컴포넌트나 리스트 상태를 `setExpandedComments` 등으로 재할당하여 갱신할지 결정해서 구현해주세요. (보통은 아이템 별 상태 변경이 더 매끄럽습니다.)

## tester 전달 메모
- 일반 유저로 로그인 후 추천 글의 댓글에 좋아요와 싫어요를 토글해보며 숫자가 잘 오르내리는지 점검해주세요.
- 관리자 권한으로 로그인 하여 동일 기능 접근 시 정상적으로 제한되는지 확인 바랍니다.
